# Production Safety Guide

This document covers what must be true before deploying this platform to production on Render.

---

## Critical Checkers (Blockers)

These **must** be fixed before launch. The SA `/api/production-readiness` endpoint will return `status: "blocked"` until they are resolved.

| Check | Requirement |
|-------|-------------|
| **Database** | `DATABASE_URL` must be set to a PostgreSQL connection string on Render. SQLite is ephemeral on Render — data is wiped on every redeploy. |
| **JWT_SECRET** | Must not be the default value (`supersecretkey_change_in_production_123456789`). Render auto-generates one via `generateValue: true` in `render.yaml`. |
| **Super Admin exists** | At least one super admin row must exist. The seed runs only if the table is empty on first deploy. |
| **Protected tables** | All core tables must exist: `restaurants`, `users`, `products`, `orders`, `customers`, `conversations`, `messages`, `subscriptions`, `super_admins`, `payment_requests`, `channels`, `bot_config`. |

---

## Warnings (Non-blocking, but important)

| Check | Recommendation |
|-------|---------------|
| **BASE_URL** | Set to your Render URL (e.g. `https://my-app.onrender.com`). Without it, Telegram webhook registration fails. |
| **OPENAI_API_KEY** | Required for AI bot replies and voice transcription. |
| **Supabase Storage** | Without `SUPABASE_URL` + `SUPABASE_SERVICE_ROLE_KEY`, menu images and payment proof uploads are stored on local disk and **will be lost after every Render redeploy**. |
| **ALLOWED_ORIGINS** | Set to your frontend domain to prevent open CORS. |

## Environment Variables — Full Reference

Declared in `render.yaml`. Set any `sync: false` vars manually in the Render dashboard.

| Variable | Source | Required |
|----------|--------|----------|
| `JWT_SECRET` | Auto-generated by Render | ✅ Critical |
| `DATABASE_URL` | Auto from `restaurant-db` binding | ✅ Critical |
| `BASE_URL` | Manual | ✅ For webhooks |
| `OPENAI_API_KEY` | Manual | ✅ For AI |
| `SUPABASE_URL` | Manual | For storage |
| `SUPABASE_SERVICE_ROLE_KEY` | Manual | For storage |

| `ENVIRONMENT` | `production` (in render.yaml) | Auto-set |
| `OPENAI_MODEL` | `gpt-4o-mini` | Auto-set |

## Migration Safety

- All schema migrations in `database.py` use `CREATE TABLE IF NOT EXISTS` and `ALTER TABLE ... ADD COLUMN`.
- **No `DROP TABLE` is ever called on protected tables.**
- Migrations are idempotent — safe to run on every startup.

### Seed Safety

| Seed | Guard |
|------|-------|
| Default subscription plans | Only runs if `subscription_plans` table is **empty** |
| Arabic plan backfill | Only updates rows where `name_ar` is empty (never overwrites SA edits) |
| Super admin | Only creates if `super_admins` table is **empty** (never overwrites password) |
| Demo restaurant | Only creates if `restaurants` table is **empty** |

## Backup & Restore

### Take a backup

```bash
# SQLite (local dev)
python scripts/backup_database.py

# PostgreSQL (set DATABASE_URL first)
DATABASE_URL=postgresql://... python scripts/backup_database.py --output-dir ./backups
```

Backup files are written to `./backups/` with a UTC timestamp. A `.sha256` checksum is written alongside.

### Verify a backup

```bash
python scripts/verify_backup.py backups/restaurant_20260426T120000Z.db
python scripts/verify_backup.py backups/restaurant_20260426T120000Z.sql
```

Checks: file size, SHA-256 checksum, SQLite integrity, required tables present.

`GET /api/production-readiness` — Super Admin only.

Returns:

```json

  "status": "ready | warnings | blocked",
  "is_production": true,
  "blockers": ["..."],
  "warnings": ["..."],
  "checks": {

    "base_url":          { "ok": true,  "value": "https://..." },
    "openai":            { "ok": true },
    "supabase_storage":  { "ok": true },
    "protected_tables":  { "ok": true,  "missing": [] },
    "super_admin":       { "ok": true },
```

Also visible in the Super Admin dashboard under **سلامة الإنتاج**.

## Deployment Checklist

Before pushing to production:

- [ ] `DATABASE_URL` bound to a Render PostgreSQL database
- [ ] `JWT_SECRET` auto-generated (verify in Render dashboard)
- [ ] `BASE_URL` set to the Render service URL
- [ ] `OPENAI_API_KEY` set
- [ ] Supabase project created, `SUPABASE_URL` + `SUPABASE_STORAGE_BUCKET_PAYMENTS` + `SUPABASE_SERVICE_ROLE_KEY` set
- [ ] `ALLOWED_ORIGINS` set to frontend domain
- [ ] `/api/production-readiness` returns `status: "ready"` or `status: "warnings"` (no blockers)
- [ ] Take a backup after first successful deploy: `python scripts/backup_database.py`
